# **NeuralForecast向け拡張 hist\_exog\_list 要件定義補完および詳細調査報告書**

## **1\. 概要と戦略的背景**

本報告書は、NeuralForecastを用いた時系列予測モデル構築プロジェクトにおいて、public.loto\_hist（ロト抽選履歴データ）から生成される過去由来外生変数（hist\_exog）の定義を拡張し、その技術的実装要件を詳細に規定するものである。ロト（Loto）のような確率的、あるいはカオス的な挙動を示す時系列データに対する特徴量エンジニアリングは、単なるラグや移動平均といった伝統的な手法だけでは不十分であり、基礎的な統計分布の変動から、非線形ダイナミクス、情報理論的複雑性、そしてドメイン特有の数論的性質に至るまで、多角的な視点からのアプローチが不可欠である。

特に、本要件定義における最重要制約である「データリーク（Data Leakage）の完全な排除」を遵守しつつ、予測精度に寄与する有意なシグナルを抽出するためには、因果的（Causal）なアルゴリズムの選定と、ストリーミング処理を前提とした実装設計が求められる。本報告では、既存の要件定義書をベースとしつつ、最新の時系列解析ライブラリ（stumpy, river, antropy, catch22 等）の調査結果に基づき、追加すべき変数のカテゴリ、具体的な計算ロジック、および実装戦略を包括的に提案する。

## **2\. 拡張特徴量エンジニアリングの理論的枠組み**

ロトの抽選結果は、形式的には独立試行の連続（ランダムウォーク）とみなされることが多いが、物理抽選機や擬似乱数生成器（PRNG）の微細なバイアス、あるいは「大数の法則」への収束過程における短期的な偏り（Local Trends）が存在する可能性がある。これらの隠れた構造を捉えるため、以下の4つの理論的枠組みに基づいて特徴量を拡張する。

### **2.1 因果的信号処理と周波数解析**

従来の高速フーリエ変換（FFT）は全期間のデータを必要とするため、そのまま適用すると未来の情報を参照するリークが発生する。これに対し、短時間フーリエ変換（STFT）やウェーブレット変換において、境界条件を厳密に制御することで、現時点までの情報のみを用いた周波数解析が可能となる1。これにより、周期性や特定の数字の出現サイクルの変化を捉える。

### **2.2 情報理論と複雑性指標**

系列の「ランダムさの質」を定量化するアプローチである。エントロピー（Entropy）や複雑性（Complexity）指標は、データが純粋なノイズであるか、あるいは決定論的なカオスを含んでいるかを判別するのに有効である。特に、Permutation EntropyやLempel-Ziv Complexityは、数値の絶対値ではなくパターンの構造に注目するため、ロトのようなデータに適している3。

### **2.3 ストリーミング・マトリックスプロファイル**

時系列データマイニングにおける最先端手法であるMatrix Profileは、部分系列（Subsequence）間の類似度を全探索し、モチーフ（頻出パターン）やディスクード（異常パターン）を特定する。stumpyライブラリの増分更新アルゴリズム（stumpi）を用いることで、計算コストを抑えつつ、リアルタイムに「過去の類似局面」との距離を特徴量化できる5。

### **2.4 ドメイン特化型数論的特徴量**

ロト特有の「迷信的」あるいは経験則的な指標（ギャップ、ホットナンバー、合計値、パリティなど）を、統計的記述子として再定義する。これらは、人間が直感的に感じる「流れ」を数値化するものであり、機械学習モデルが人間心理に起因するバイアス（もし選択式であれば）や、物理的な偏りを学習する助けとなる7。

**3\. 追加提案：詳細特徴量カテゴリと計算ロジック**

既存の要件（ラグ、移動平均等）に加え、以下のカテゴリを実装することを提案する。これらはPythonエコシステム内の専門ライブラリを活用することで、効率的かつ正確に計算可能である。

### **3.1 情報理論・複雑性指標 (Information Theory & Complexity)**

このカテゴリの特徴量は、時系列の不規則性や予測困難性を測定する。ロトの抽選結果が「どの程度ランダムか」あるいは「何らかの構造的崩れがあるか」を検知するセンサーとして機能する。

| 特徴量サブカテゴリ | 具体的な指標 | 計算ロジックと期待される効果 | 推奨ライブラリ |
| :---- | :---- | :---- | :---- |
| **順列エントロピー** | **Permutation Entropy** | 窓内の値を順序パターン（例: $x\_t \< x\_{t-1} \< x\_{t-2}$）に変換し、その出現分布のエントロピーを計算する。ノイズに対して堅牢であり、振幅ではなく「形状の複雑さ」を捉える4。 | antropy, tsfresh |
| **サンプルエントロピー** | **Sample Entropy (SampEn)** | 系列内で類似したパターンが繰り返される頻度を対数尤度で評価する。生体信号分析で標準的に使われ、カオス的挙動の定量化に優れる10。計算コストが高いため、短い窓での適用や高速アルゴリズムが必要。 | antropy, nolds |
| **スペクトルエントロピー** | **Spectral Entropy** | パワースペクトル密度（PSD）を確率分布とみなし、そのシャノンエントロピーを計算する。信号がホワイトノイズに近いほど値が高く、特定の周波数成分が支配的であるほど低くなる4。 | antropy, scipy |
| **アルゴリズム的複雑性** | **Lempel-Ziv Complexity** | 系列を2値化（0/1）した後、LZ77圧縮アルゴリズムを用いて「新しい部分文字列」の発生数をカウントする。パターンの反復性を評価し、ランダム性の検定に利用される12。 | antropy, tsfresh |
| **信号特性** | **Hjorth Parameters** | Activity（分散）、Mobility（パワースペクトルの重心周波数近似）、Complexity（周波数の変化率）の3要素。計算が高速でありながら、信号の物理的特性を捉える能力が高い4。 | antropy |
| **特異値分解エントロピー** | **SVD Entropy** | 窓内のデータを埋め込み行列（Trajectory Matrix）に変換し、その特異値分布のエントロピーを計算する。データの次元数や自由度を示唆する4。 | antropy |

**実装上の留意点:** エントロピー計算は計算負荷が高いため、antropyのようなNumbaで最適化されたライブラリの使用が必須である。また、窓幅（Window Size）の選定が結果に大きく影響するため、複数の窓幅（例: 短期、中期）で計算し、それらを別の特徴量として扱うことが望ましい。

### **3.2 部分系列解析とマトリックスプロファイル (Matrix Profile)**

時系列中の「部分的な形状」の類似性を評価する。これは、過去の特定の期間と現在の挙動が似ているかどうかを判断するために極めて強力な手法である。

| 特徴量サブカテゴリ | 具体的な指標 | 計算ロジックと期待される効果 | 推奨ライブラリ |
| :---- | :---- | :---- | :---- |
| **最近傍距離** | **1-NN Distance (Min Dist)** | 現在のウィンドウ（部分系列）と、過去の全ての部分系列との間の最小ユークリッド距離。値が小さい場合、過去に酷似したパターンが存在すること（Motif）を示し、大きい場合は未知のパターン（Anomaly/Discord）であることを示す13。 | stumpy |
| **最近傍インデックス** | **Matrix Profile Index** | 最も似ている過去の部分系列が「いつ」発生したかを示すインデックス。これをそのまま特徴量とするほか、このインデックスが指す時点の y の値を「類似ラグ」として参照することも有効である6。 | stumpy |
| **レジーム変化** | **Regime Change / FLUSS** | Matrix Profileを用いて、時系列の構造的な変化点（Regime Shift）を検出するスコア。ロトの抽選アルゴリズムや物理的条件が変化した可能性を示唆する15。 | stumpy |

**リーク防止の実装:** 標準的なMatrix Profile（stump）は全期間の総当たり計算を行うため、未来参照のリスクがある。必ず **stumpi (Incremental Matrix Profile)** を使用し、新しいデータ点が到着するたびにプロファイルを更新するストリーミング方式を採用しなければならない5。

### **3.3 ストリーミング異常検知とオンライン学習 (Online Anomaly Detection)**

ロトの結果において、統計的にあり得ない偏りや異常な値をリアルタイムで検知し、その「異常スコア」を特徴量として利用する。バッチ学習ではなく、逐次更新型のアルゴリズムを用いることで、概念ドリフト（分布の経時変化）に追従する。

| 特徴量サブカテゴリ | 具体的な指標 | 計算ロジックと期待される効果 | 推奨ライブラリ |
| :---- | :---- | :---- | :---- |
| **木ベース異常検知** | **Half-Space Trees (HST)** | ストリーミングデータ専用の異常検知アルゴリズム。空間をランダムに分割し、データ点が疎な領域に落ちる頻度から異常度を算出する。高速かつメモリ効率が良い17。 | river, pysad |
| **確率的検知** | **LODA (Lightweight On-line Detector of Anomalies)** | 多数のランダムなヒストグラムを用いて確率密度を推定し、低密度領域の観測値を異常とする。特徴間の相関が希薄な場合でも有効17。 | pysad |
| **オンライン分位点** | **Incremental Quantile / T-Digest** | データの正確な分位点（Percentile）を逐次更新する。現在の値が過去の分布のどの位置（ランク）にあるかを0-1で正規化して出力する19。 | river |
| **予測誤差スコア** | **Predictive Anomaly Score** | 単純なオンライン予測モデル（例: 線形回帰）を走らせ、その予測誤差の大きさを異常スコアとする。 | river |

**実装戦略:** river や pysad は learn\_one (学習) と score\_one (スコアリング) のメソッドを持つ。特徴量生成時は、まず score\_one で現在の値の異常度を計算し、その後に learn\_one でモデルを更新するという順序を厳守することで、リークを完全に回避できる。

### **3.4 因果的周波数解析 (Causal Frequency Analysis)**

波形解析の観点から、周期性や振動成分を抽出する。

| 特徴量サブカテゴリ | 具体的な指標 | 計算ロジックと期待される効果 | 推奨ライブラリ |
| :---- | :---- | :---- | :---- |
| **短時間フーリエ変換** | **Causal STFT** | スライディングウィンドウに対しFFTを適用するが、窓の「右端」を現時点とする。未来のデータを含めないパディング（ゼロ埋め等）を行い、スペクトルの重心やピーク周波数を抽出する20。 | scipy.signal |
| **因果的ウェーブレット** | **Causal Wavelet Energy** | 過去データのみを参照する境界条件を用いた離散ウェーブレット変換（DWT）。各分解レベル（Scale）におけるエネルギー（係数の二乗和）を計算し、多重解像度での変動強度を捉える2。 | PyWavelets |
| **ピーク解析** | **Number of Peaks / CWT Peaks** | 窓内におけるピークの数や幅。連続ウェーブレット変換（CWT）を用いて、特定のスケールでのピーク出現を検知する23。 | scipy.signal |

ロトの核心である「ランダム性」そのものを検定する指標。

| :---- | :---- | :---- | :---- |
| **連検定** | **Runs Test Z-Statistic** | 値が基準（中央値等）を上回る/下回る連続（Run）の数を数え、理論的な期待値からの乖離をZスコア化する。過剰なトレンドや交互作用（Mean Reversion）を検知する24。 | statsmodels |

### **3.6 ドメイン特化型数論的特徴量 (Loto Specific)**

数字そのものの性質や、組み合わせ論的な特徴に着目する。これらは y が単なる数値ではなく、セット球や当選番号のセットである場合に特に意味を持つ。

| 特徴量サブカテゴリ | 具体的な指標 | 計算ロジックと期待される効果 | 実装方針 |
| :---- | :---- | :---- | :---- |
| **ギャップ解析** | **Gap Statistics** | 各数字が最後に出現してからの経過回数（Gap）。「ハマり」数字や「連発」数字を特定する。全数字のGapの平均、分散、最大値などを要約統計量とする9。 | カスタム (Pandas) |
| **ホット/コールド** | **Decay Frequency Score** | 単純な頻度ではなく、過去に行くほど重みが指数的に減少する加重頻度（EWM Frequency）。直近の「勢い」を数値化する7。 | カスタム / tsfresh |
| **数字の性質** | **Sum / Root / Parity** | 当選数字の合計、数字根（各桁の和の再帰）、奇数偶数の比率、素数・合成数の割合など。分布の偏りを捉える30。 | カスタム |

### **4.1 パイプライン構成案**

実装にあたっては、以下のフローで処理を行うパイプラインを構築する。

1. **Snapshot Resolution**: public.loto\_hist から最新のスナップショットを取得し、時系列順に整列する。  
2. **Sequential Processing (Stateful)**:  
   * river や stumpi などのステートフルなモデルを初期化する。  
   * データを古い順に1行ずつ（またはミニバッチで）処理し、特徴量を計算してからモデルを更新する（Predict-then-Learn）。  
3. **Rolling Window Processing (Stateless)**:  
   * pandas.rolling や tsflex を用い、固定窓での統計量（エントロピー、FFT等）を計算する。この際、shift(1) を適用し、現在の y が計算に含まれないようにする。  
4. **Integration & Formatting**:  
   * 生成された全特徴量を結合し、欠損値（初期窓分）を処理（設定により0埋めまたは削除）する。  
   * 最終的に public.loto\_hist\_exog のスキーマに合わせて出力する。

**5\. 品質保証とリーク検証プロセス**

要件定義書の「10. 品質要件」を強化し、特にリーク検証における具体的なテスト手法を定める。

### **5.1 リーク検証の自動テスト (Leakage Detection Unit Tests)**

単なる目視確認ではなく、以下のロジックを用いた自動テスト（Unit Test）を実装する。

* **Perturbation Test (摂動テスト)**:  
  1. ある時点 $t$ の特徴量ベクトル $F\_t$ を計算する。  
  2. 元データの時点 $t+1$ 以降の y にランダムなノイズを加える、あるいは値を変更する。  
  3. 再度 $t$ の特徴量を計算し、$F'\_t$ とする。  
  4. $F\_t$ と $F'\_t$ が完全に一致することを確認する（浮動小数点の誤差許容範囲内で）。一致しない場合、未来のデータが $t$ の計算に混入している（リーク）と判定する。  
* **Shift Verification**:  
  * 全ての特徴量生成関数の入力において、明示的に .shift(periods=1) が適用されているか、あるいはインデックス操作により $t$ を除外しているかを静的解析またはラップ関数で強制する。

### **5.2 統計的妥当性チェック**

* **相関チェック**: 生成された特徴量間で相関係数を計算し、著しく高い（\>0.99）ペアが存在する場合は警告を出す（計算リソースの無駄であるため）。catch22のような削減セットの利用を推奨する根拠とする。  
* **定常性チェック**: 生成された hist\_exog に対しADF検定を行い、非定常な特徴量が含まれていないか確認する。非定常な変数はモデルの学習を不安定にする可能性がある。

  matrix\_profile:  
    enabled: true  
    library: "stumpy"  
    windows:  \# Subsequence length (m)

## **7\. 結論**

本調査により、NeuralForecast向けの hist\_exog として、従来の時間領域統計量に加え、**情報理論（エントロピー）**、**部分系列類似性（Matrix Profile）**、**ストリーミング異常検知** の3領域を重点的に強化すべきであることが判明した。これらの特徴量は、ロトのような確率的プロセスの隠れた構造や偏りを捉える可能性があり、かつ stumpy や river などの最新ライブラリを用いることで、リークを回避しつつ効率的に実装可能である。特に **Permutation Entropy** と **Incremental Matrix Profile** は、ノイズへの頑健性と計算効率のバランスが良く、優先的に実装すべき「高付加価値特徴量」として推奨する。
